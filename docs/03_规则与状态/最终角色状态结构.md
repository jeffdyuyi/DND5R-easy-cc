# 最终角色状态结构

## 1. 文档目标

本文件定义角色创建器在“选择完成并经过统一计算后”应保存和消费的最终角色状态结构。  
它是角色创建流程、角色卡展示、法术页、装备页、负重页、导出备份和后续再编辑的共同数据基准。

本文件的定位：

- 面向产品：明确角色卡最终要显示哪些内容
- 面向程序：明确最终角色对象至少要有哪些字段
- 面向数据：明确职业、背景、种族、专长、法术、装备最终要汇总到哪里

## 2. 设计原则

### 2.1 最终角色状态不是原始选择集合

最终状态必须同时包含：

- 原始选择结果
- 自动统合后的计算结果
- 用于展示的结构
- 用于追溯来源的结构

### 2.2 最终角色状态必须可复算

保存后的角色对象应当支持：

- 重新打开继续编辑
- 重新从原始选择重算
- 从最终结果回溯来源

因此不能只保存纯展示文本。

### 2.3 最终角色状态必须适配角色卡

结合参考角色卡，最终状态必须能覆盖这些展示区域：

- 基本身份区
- 六维属性与豁免
- 技能区
- HP/HD/死亡豁免
- AC/先攻/速度/体型/被动察觉
- 种族特性、职业特性、专长
- 武器与攻击
- 法术区
- 装备与同调魔法物品
- 语言、背景故事、个性、阵营
- 钱币、财富、负重、背包

## 3. 顶层结构建议

建议最终角色状态至少包含以下顶层块：

```ts
type FinalCharacterState = {
  meta: CharacterMeta
  identity: CharacterIdentity
  progression: CharacterProgression
  abilities: CharacterAbilities
  combat: CharacterCombat
  proficiencies: CharacterProficiencies
  traits: CharacterTraits
  spells: CharacterSpellState
  inventory: CharacterInventory
  carrying: CharacterCarryingState
  notes: CharacterNotes
  sources: CharacterSourceTrace
  computed: CharacterComputedState
}
```

说明：

- `identity` 保存“你是谁”
- `progression` 保存“你现在几级、什么职业”
- `abilities/combat/proficiencies/spells/inventory` 保存可直接展示的主结果
- `sources` 保存来源追踪
- `computed` 保存统一结算产物

## 4. `meta` 元信息

```ts
type CharacterMeta = {
  id: string
  version: string
  createdAt: string
  updatedAt: string
  ruleset: '5e_2024'
  buildMode?: 'player' | 'gm'
  status?: 'draft' | 'playable' | 'archived'
}
```

字段说明：

- `id`: 角色唯一 ID
- `version`: 角色状态结构版本
- `createdAt/updatedAt`: 保存与更新时刻
- `ruleset`: 当前规则版本
- `status`: 角色当前状态

---

## 5. `identity` 身份信息

这部分对应角色卡上的基本身份栏与人物背景栏。

```ts
type CharacterIdentity = {
  name: string
  playerName?: string
  portraitUrl?: string
  alignment?: string
  faith?: string
  homeland?: string
  age?: string
  height?: string
  weightText?: string
  gender?: string
  appearance?: string
  personalityTraits?: string[]
  ideals?: string[]
  bonds?: string[]
  flaws?: string[]
  backstory?: string
}
```

说明：

- 这部分允许更多文本化内容
- 不参与核心规则结算
- 但必须能完整支撑参考角色卡的身份展示区

---

## 6. `progression` 构建进度与构筑身份

这部分是角色创建器最关键的结构之一。

```ts
type CharacterProgression = {
  level: number
  experience?: number
  proficiencyBonus: number
  classes: CharacterClassEntry[]
  primaryClassId?: string
  primaryClassName?: string
  subclassId?: string
  subclassName?: string
  speciesId?: string
  speciesName?: string
  subraceId?: string
  subraceName?: string
  backgroundId?: string
  backgroundName?: string
}

type CharacterClassEntry = {
  classId: string
  className: string
  source: string
  level: number
  subclassId?: string
  subclassName?: string
}
```

说明：

- 采用 `classes[]` 而不是单一职业字段，确保兼容兼职
- `primaryClass*` 是为了方便 UI 展示
- `species/subrace/background` 用于快速定位角色构筑来源

## 7. `abilities` 属性状态

```ts
type CharacterAbilities = {
  scores: AbilityScoreMap
  modifiers: AbilityModifierMap
  saveModifiers: AbilityModifierMap
  saveProficiencies: string[]
  heroInspiration?: boolean
}

type AbilityScoreMap = {
  str: number
  dex: number
  con: number
  int: number
  wis: number
  cha: number
}

type AbilityModifierMap = {
  str: number
  dex: number
  con: number
  int: number
  wis: number
  cha: number
}
```

说明：

- `scores` 是最终属性值
- `modifiers` 是最终调整值
- `saveModifiers` 是豁免数值，已经考虑熟练等修正
- `saveProficiencies` 用于角色卡勾选展示

## 8. `combat` 战斗状态

这部分对应参考角色卡中的战斗主区。

```ts
type CharacterCombat = {
  armorClass: number
  armorClassBreakdown: string[]
  initiative: number
  speed: CharacterSpeedState
  size: string
  passivePerception: number
  hitPoints: CharacterHitPointState
  hitDice: CharacterHitDiceState
  deathSaves: CharacterDeathSaveState
  resistances: string[]
  immunities: string[]
  vulnerabilities: string[]
  conditions: string[]
}

type CharacterSpeedState = {
  walk: number
  fly?: number
  swim?: number
  climb?: number
  burrow?: number
  adjustedWalk?: number
}

type CharacterHitPointState = {
  current: number
  max: number
  temp: number
}

type CharacterHitDiceState = {
  total: string[]
  spent: string[]
}

type CharacterDeathSaveState = {
  success: number
  failure: number
}
```

说明：

- `armorClassBreakdown` 是强制要求，不能只保存一个数字
- `adjustedWalk` 用于负重、状态、特性等影响后的移动速度

## 9. `proficiencies` 熟练与训练

```ts
type CharacterProficiencies = {
  skills: SkillProficiencyEntry[]
  tools: string[]
  weapons: string[]
  armors: string[]
  languages: string[]
}

type SkillProficiencyEntry = {
  skill: string
  proficient: boolean
  expertise?: boolean
  modifier: number
}
```

说明：

- `skills` 要保存最终数值而不是只保存是否熟练
- `languages` 单独放在熟练区，便于展示和管理

## 10. `traits` 特性与能力

这部分对应角色卡中的种族特性、职业特性、专长、特殊能力等区域。

```ts
type CharacterTraits = {
  speciesTraits: CharacterTraitEntry[]
  classFeatures: CharacterTraitEntry[]
  subclassFeatures: CharacterTraitEntry[]
  backgroundFeatures: CharacterTraitEntry[]
  feats: CharacterFeatEntry[]
  specialAbilities: CharacterTraitEntry[]
}

type CharacterTraitEntry = {
  id?: string
  name: string
  source: string
  description: string
  category?: string
}

type CharacterFeatEntry = {
  featId: string
  name: string
  source: string
  category: string
  description: string
  repeatable?: boolean
}
```

说明：

- 这里是角色卡展示结构
- 其真正来源追踪应放到 `sources`

## 11. `spells` 法术状态

这部分要同时覆盖角色卡中的法术速查区和法术书区。

```ts
type CharacterSpellState = {
  spellcastingAbilities: SpellcastingAbilityEntry[]
  spellSaveDCs: SpellSaveDcEntry[]
  spellAttackBonuses: SpellAttackBonusEntry[]
  spellSlots: SpellSlotEntry[]
  cantrips: CharacterSpellEntry[]
  knownSpells: CharacterSpellEntry[]
  preparedSpells: string[]
  alwaysPreparedSpells: string[]
  grantedSpells: GrantedSpellEntry[]
}

type SpellcastingAbilityEntry = {
  sourceId: string
  sourceType: 'class' | 'subclass' | 'species' | 'background' | 'feat' | 'item'
  ability: 'str' | 'dex' | 'con' | 'int' | 'wis' | 'cha'
}

type SpellSaveDcEntry = {
  sourceId: string
  dc: number
}

type SpellAttackBonusEntry = {
  sourceId: string
  bonus: number
}

type SpellSlotEntry = {
  level: number
  max: number
  used: number
}

type CharacterSpellEntry = {
  spellId: string
  name: string
  source: string
  spellLevel: number
  school?: string
  castingTime?: string
  range?: string
  duration?: string
  concentration?: boolean
  ritual?: boolean
  materials?: boolean
}

type GrantedSpellEntry = {
  spellId: string
  grantType: 'known' | 'prepared' | 'always_prepared' | 'free_cast' | 'choice_pool'
  sourceId: string
  sourceType: 'class' | 'subclass' | 'species' | 'background' | 'feat' | 'item'
  notes?: string
}
```

说明：

- `knownSpells` 是最终可查看的法术列表
- `grantedSpells` 用于保留授予语义
- `preparedSpells` 和 `alwaysPreparedSpells` 分开，避免混淆

## 12. `inventory` 装备、背包、钱币、同调

这部分要兼容参考角色卡里的装备、背包、同调魔法物品、钱币、总财富。

```ts
type CharacterInventory = {
  equippedArmorId?: string
  equippedShieldId?: string
  equippedWeaponIds: string[]
  equipment: CharacterInventoryItem[]
  backpacks: CharacterContainerEntry[]
  attunedItemIds: string[]
  currency: CharacterCurrency
  wealth: CharacterWealthState
}

type CharacterInventoryItem = {
  itemId: string
  name: string
  source: string
  category: string
  quantity: number
  equipped?: boolean
  attuned?: boolean
  weightLb?: number
  costText?: string
  notes?: string
}

type CharacterContainerEntry = {
  id: string
  name: string
  itemIds: string[]
}

type CharacterCurrency = {
  cp: number
  sp: number
  ep: number
  gp: number
  pp: number
}

type CharacterWealthState = {
  totalValueGp: number
  coinWeightLb: number
}
```

说明：

- `equipment` 是实际物品明细
- `backpacks` 用于支持背包、次元袋等容器视图
- `wealth.totalValueGp` 是统一财富总值

## 13. `carrying` 负重与载重状态

本块是强制要求，必须对接负重规则。

```ts
type CharacterCarryingState = {
  sizeMultiplier: number
  carryCapacityLb: number
  pushDragLiftLb: number
  encumberedThresholdLb?: number
  heavilyEncumberedThresholdLb?: number
  equipmentWeightLb: number
  inventoryWeightLb: number
  coinWeightLb: number
  currentCarryWeightLb: number
  encumbranceRatio: number
  encumbranceState: 'normal' | 'encumbered' | 'heavily_encumbered' | 'over_capacity'
  speedPenaltyFt?: number
  disadvantageFlags?: string[]
}
```

说明：

- `equipmentWeightLb`、`inventoryWeightLb`、`coinWeightLb` 必须能分开显示
- `encumbranceRatio` 用于进度条和比例展示
- `disadvantageFlags` 用于记录变体负重产生的检定/攻击/豁免劣势

## 15. `sources` 来源追踪

这部分是实现“自动统合但可追溯”的关键。

```ts
type CharacterSourceTrace = {
  selectedIds: string[]
  grants: CharacterGrantTrace[]
  appliedAdjustments: CharacterAdjustmentTrace[]
}

type CharacterGrantTrace = {
  sourceId: string
  sourceType: 'class' | 'subclass' | 'species' | 'background' | 'feat' | 'item'
  targetType: 'feat' | 'spell' | 'proficiency' | 'ability_bonus' | 'item' | 'language' | 'armor_class'
  targetId?: string
  targetName?: string
  mode: 'fixed' | 'choice'
  notes?: string
}

type CharacterAdjustmentTrace = {
  sourceId: string
  target: string
  value: string
  notes?: string
}
```

说明：

- `selectedIds` 是用户明确选择过的条目
- `grants` 用于追踪嵌套授予链
- `appliedAdjustments` 用于追踪数值修正来源

## 16. `computed` 统一计算结果

该块用于存放计算模块最终输出的标准结果，供角色卡直接消费。

```ts
type CharacterComputedState = {
  isValid: boolean
  completion: CharacterCompletionState
  summaries: CharacterSummaryState
}

type CharacterCompletionState = {
  hasClass: boolean
  hasSpecies: boolean
  hasBackground: boolean
  hasAbilities: boolean
  hasIdentity: boolean
}

type CharacterSummaryState = {
  classSummary?: string
  speciesSummary?: string
  backgroundSummary?: string
  featSummary?: string
  spellSummary?: string
  loadSummary?: string
}
```

说明：

- `computed` 不是为了替代其他数据块
- 它是最终给 UI 的快捷结果层

## 17. 最小可保存结构与最小可展示结构

### 17.1 最小可保存结构

若角色只完成了部分创建流程，至少也应能保存：

- `meta`
- `identity.name`
- `progression`
- `abilities.scores`
- `sources.selectedIds`

### 17.2 最小可展示结构

一个可以进入角色卡主视图的角色，至少应具备：

- 姓名
- 等级
- 职业
- 种族
- 背景
- 六维属性与调整值
- HP
- AC
- 速度
- 熟练加值

## 19. 实现建议

建议最终在代码层把角色对象分为两类：

### 19.1 编辑态角色对象

用于向导步骤、用户选择、待确认池、临时选择。

### 19.2 计算态角色对象

用于：

- 角色卡展示
- 规则结算
- 导出存档
- 重新打开时的状态恢复

其中本文件定义的是“计算态角色对象”的目标结构。

---

## 20. 结论

最终角色状态结构必须满足三件事：

1. 能完整映射角色卡展示字段
2. 能支撑自动统合和递归授予
3. 能从选择状态稳定复算

因此，最终角色状态不能只是一组“最终文本”，而必须是一份带有：

- 选择结果
- 计算结果
- 来源追踪
- 展示字段

的统一角色对象。
