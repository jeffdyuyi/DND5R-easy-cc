# 缺失参考文档清单

## 1. 现状判断

当前仓库已经具备三类重要基础文档：

- 数据层：清洗方案、统一约束规范、专题执行文档
- 规则层：角色构建计算口径、最终角色状态、编辑态与计算态边界
- 参考层：流程指南、专题概览、术语映射与完整性报告

但从“做出一个可交付工具”来看，仍缺少把这些规范转成可执行需求、可拆解任务和可验证交付件的桥梁文档。

截至当前版本，以下桥梁文档已经补齐：

- `MVP范围与PRD.md`
- `分步建卡验收标准.md`
- `领域模型与类型契约.md`
- `规则计算引擎设计.md`
- `测试策略.md`

因此本清单后续应主要用于跟踪“仍待补齐的文档”，而不是重复列出已完成项。

### 2.3 分步向导交互与验收标准

现有流程文档更像思路说明，还缺每一步的：

- 输入
- 输出
- 校验规则
- 失败提示
- 可回退逻辑

建议新增：

- `docs/01_产品与流程/分步建卡验收标准.md`

需要把玩家端与主持人端页面关系说清楚：

- 角色卡详情

- 装备页
- 主持人资源管理后台

- `docs/01_产品与流程/信息架构与页面地图.md`

## 3. 从程序员视角缺少的文档

### 3.1 实际工程初始化说明

`项目技术总览` 描述了 React + Vite 工程结构，但当前仓库未发现：

- `package.json`
- `vite.config.ts`
- `App.tsx`
- `index.tsx`

因此当前缺少真正的工程启动文档。

建议新增：

- `docs/02_数据工程/工程初始化与目录约定.md`

### 3.2 领域模型与 TypeScript 类型契约

现有状态结构文档主要说明目标对象，但还缺：

- `staging-data` 到应用模型的转换契约
- 前端消费模型类型定义
- GM 编辑态模型定义
- 导入导出模型与版本号

建议新增：

- `docs/03_规则与状态/领域模型与类型契约.md`

### 3.3 规则计算引擎设计

当前已有“计算口径”，但还缺工程级设计：

- 计算入口
- 中间层数据结构
- effect resolver 机制
- grant 递归解析器
- 冲突解决优先级
- 重算时机

建议新增：

- `docs/03_规则与状态/规则计算引擎设计.md`

### 3.4 分步向导状态机设计

分步式角色卡创建最容易失控的地方不是 UI，而是状态流转。

需要文档明确：

- 步骤状态机
- pending choices 结构
- skip / back / resume 逻辑
- 自动插入步骤的机制

建议新增：

- `docs/03_规则与状态/分步向导状态机设计.md`

### 3.5 数据校验与导入流水线

还需要明确：

- 原始数据如何进入 staging
- staging 如何验证
- 如何生成应用消费数据
- 构建时如何跑校验

建议新增：

- `docs/02_数据工程/数据校验与导入流水线.md`

### 3.6 测试策略

目前没有看到对以下内容的测试说明：

- 规则计算单测
- 数据快照测试
- 角色创建流程集成测试
- GM CRUD 回归测试

建议新增：

- `docs/03_规则与状态/测试策略.md`

### 3.7 持久化与版本迁移策略

如果用本地存档、导出 JSON、战役配置快照，就必须定义：

- 存档 schema version
- 升级迁移规则
- 不兼容数据处理

建议新增：

- `docs/03_规则与状态/存档与迁移策略.md`

### 3.8 主持人资源管理边界

“主持人资源管理工具”不仅是 CRUD，还要定义：

- 哪些字段可编辑
- 哪些字段只能来源于清洗数据
- 审核态和发布态怎么区分

建议新增：

- `docs/01_产品与流程/主持人资源管理需求说明.md`
