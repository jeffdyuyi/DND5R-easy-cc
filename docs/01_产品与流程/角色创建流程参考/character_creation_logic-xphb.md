# D&D 2024 (XPHB) 角色创建工具实现逻辑指南

本指南旨在为开发一套高扩展性、数据驱动的角色创建工具提供核心“技术底座”。其设计目标是优先建立通用的计算公式，并辅以具体的数据库特例作为参考。

---

## 1. 核心流程与数据映射 (Process Mapping)

开发工具时，UI 步骤应与以下数据路径保持一致：

| 步骤 | 关键动作 | 核心关键词 | 数据源 (推荐) |
| :--- | :--- | :--- | :--- |
| **1. 初始** | 选择种族 | `Species`, `Traits`, `Senses` | `03_种族/` |
| **2. 起源** | 选择背景 | `Background`, `Origin Feats`, `ASI` | `02_背景/` |
| **3. 职业** | 选择职业 | `Class`, `Hit Dice`, `Proficiency` | `01_职业与子职业/` |
| **4. 属性** | 分配属性值 | `Ability Scores`, `Modifiers` | `10_属性与技能/` |
| **5. 派生** | 计算防御与感官 | `AC`, `HP`, `Initiative`, `Senses` | `05_物品/`, `12_规则释义/` |
| **6. 细节** | 熟练与法术 | `Proficiency`, `Spell Slots`, `PB` | 模型内部计算 |

## 2. 核心数学概念算式 (Conceptual Formulas)

本部分定义的算式应视为 UI 的**基础计算类 (Utility Classes)**。

### 2.1 熟练与属性底座
- **属性调整值 (Modifier)**: `floor((属性值 - 10) / 2)`。
- **熟练加值 (PB)**: 取决于**角色总等级**（1-4级:+2, 5-8级:+3, 以此类推）。
- **总加值 (General)**: `属性调整值 + PB (如果你熟练) + PB (如果你精通)`。

### 2.2 防御与生存 (Defense)
- **护甲等级 (AC) 通用逻辑**:
    - **基础 AC**: 默认为 `10 + 敏捷调整值`。
    - **着装护甲**: `护甲底值 + (敏捷加值上限) + 盾牌加值`。
    - **替换公式**: 当存在多项 AC 算式时（如天生护甲或职业特性），玩家**选其一且不可叠加**（盾牌除外，除非另有说明）。
- **生命值 (HP) 通用逻辑**:
    - **1 级**: `职业生命骰最大值 + 体质调整值`。
    - **后续等级**: `(现有 HP) + (生命骰平均值 或 掷骰结果) + 体质调整值`。

### 3.3 战斗与检定 (Combat & Checks)
- **先攻 (Initiative)**: `敏捷调整值 + 特殊修正`。
- **豁免检定 (Saving Throws)**: `属性调整值 + PB (如果熟练)`。
- **攻击检定 (Attack Roll)**: `基于属性调整值 + PB (如果熟练)`。
- **法术 DC**: `8 + PB + 施法属性调整值`。
- **法术攻击加值**: `PB + 施法属性调整值`。
- **被动感官**: `10 + 感官调整值 + PB (如果熟练) + 优势(+5)/劣势(-5)`。

## 3. 2024 (XPHB) 核心变动逻辑 (Revision Notes)

- **ASI (属性提升)**: `+2/+1` 或 `+1*3` 固定由 **背景 (Background)** 提供。
- **1 级起始专长**: 必须通过背景配置一个 **起源专长 (Origin Feat)**。
- **装备来源**: `最终装备 = 职业 (A/B二选一) + 背景 (A/B二选一)`。
- **语言扩展**: 初始通常为“通用语 + 2 门自选或推荐语言”。

## 4. 进阶系统：施法与兼职 (Advanced Logic)

### 4.1 施法系统
- **职业法术位**: 单职业直接查数据表。
- **兼职计算**: `Effective Caster Level (L) = 全职等级 + floor(半职/2, 向上取整) + floor(三职/3)`。
    > [!IMPORTANT]
    > 2024 版（XPHB）对半职等级（Paladin/Ranger）在兼职计算时通常采用“向上取整”原则，开发者需在逻辑中设置 Edition 开关。
- **额外来源法术**: 种族/专长直接赋予的法术通常具有“常驻准备”且“免费施展一次 + 消耗位施展”双重属性。

### 4.2 兼职门槛
- 角色必须同时满足 **既有职业** 以及 **目标职业** 的属性门槛值（通常为 13）。

## 6. 可选系统：塔莎的自定义起源 (TCoE Customization)

针对仍在使用 2014 版种族数据但希望获得 2024 式灵活性的玩家，系统应支持以下逻辑：

- **属性值交换 (AS Swap)**: 允许将种族固有的 `ASI` 自由分配。
    - *约束*: 不能叠加到同一属性，且上限为 20。
- **熟练项替换 (Proficiency Swap)**:
    - **技能 -> 技能** (任选)。
    - **护甲 -> 武器/工具**。
    - **武器 -> 武器/工具**。
    - **工具 -> 工具/武器**。
- **自定义血统 (Custom Lineage)**: 提供一个“空白模板”种族，允许玩家自行定义体型、专长和技能/感官。
