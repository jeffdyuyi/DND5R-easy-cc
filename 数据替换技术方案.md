# DND5R-easy-cc 数据替换与兼容性转换技术方案

## 目标
将 `中文data/` 目录下的数据逐步替换为 `data/` 目录下的数据源，实现数据调用的平滑过渡，并确保现有功能不受影响。

## 现状分析
- `data/` 目录下的数据为 TypeScript/JavaScript 格式，结构已适配项目代码，直接可用。
- `中文data/` 目录下为 Markdown、JSON 等原始或半结构化数据，未直接适配代码调用。
- 直接替换会导致类型不兼容、接口不统一、功能异常。

## 技术步骤

### 1. 数据结构梳理与映射
- 梳理 `data/` 下各数据文件的结构（如 class、background、species、spell 等），明确字段、类型、导出方式。
- 对比 `中文data/` 下同类数据的原始结构，建立字段映射表。
- 记录特殊字段、嵌套结构、枚举值等差异。

### 2. 编写数据转换脚本
- 在 `scripts/` 或根目录下新建数据转换脚本（如 `convert_zh_data.ts`）。
- 脚本功能：
  - 读取 `中文data/` 下的原始数据（支持 Markdown/JSON 等格式）。
  - 按照映射表转换为 `data/` 目录下的 TypeScript/JavaScript 结构。
  - 自动生成/覆盖到 `data/` 目录下的目标文件（如 `data-classes.ts`、`backgrounds/*.ts` 等）。
  - 保留原有 `data/` 数据，生成新文件时加后缀（如 `-zh.ts`），便于回滚和对比。

### 3. 兼容性适配与数据切换
- 在项目中引入配置项（如 `config.ts`），用于切换数据源（原版/中文）。
- 修改数据导入逻辑，支持根据配置动态选择数据文件（如 `import classes from './data/data-classes-zh'`）。
- 保证类型一致性，必要时为中文数据生成类型声明（`.d.ts`）。

### 4. 自动化与校验
- 转换脚本支持批量处理、日志输出、异常捕获。
- 增加数据校验逻辑，确保字段完整性、类型正确性。
- 可选：编写单元测试，验证新旧数据源下主要功能一致。

### 5. 逐步替换与优化
- 先对单一数据类型（如职业、种族）试点转换，验证无误后逐步推广。
- 保留原有 `data/` 目录，待全部验证通过后再考虑彻底替换。
- 持续优化转换脚本，支持增量更新、字段补全、格式美化等。

## 推荐目录结构
```
/scripts
  convert_zh_data.ts
/data
  data-classes.ts
  data-classes-zh.ts
  ...
/config.ts
```

## 注意事项
- 不直接覆盖原有 `data/` 数据，所有新数据以 `-zh` 后缀区分。
- 保证转换脚本幂等性，可多次运行。
- 关注数据字段兼容性，必要时补充类型声明。
- 逐步切换，随时可回滚。

## 后续优化建议
- 统一数据格式，逐步淘汰 Markdown/JSON 原始数据。
- 建立数据校验与自动化测试流程。
- 支持多语言数据切换，提升国际化能力。
