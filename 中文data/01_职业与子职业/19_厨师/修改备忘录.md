# 19_厨师 模块化与无损修复工程 - 修改备忘录

## 1. 核心架构重构
- **主类文件标准化**：将 `01_主类/厨师.json` 从单一对象重构为标准的 5etools JSON 格式（包含 `_meta`, `class`, `classFeature`）。
- **特性分离**：将所有职业特性（Class Features）从 class 对象中分离，移至 `classFeature` 数组，并注入了原始 JSON 中的完整文本内容，修复了之前仅有标题无内容的问题。
- **子职业 1-Way Binding**：子职业文件统一采用 `subclass` + `subclassFeature` 结构，通过 `subclassFeatures` 引用实现 100% 数据解耦，确保 Character Builder 正确解析。

## 2. 无损数据填充 (Lossless Repair)
- **核心特性补全**：注入了 `厨包`、`餐食`、`醒神嗅盐`、`圆肚回弹`、`食之魂灵` 等所有核心特性的规则正文，严禁任何形式的删改。
- **可选特性提取**：创建了 `03_可选特性/厨师特性.json`，完整提取并录入了全部 30 余种“餐食 (Morsels)”，包括：
  - 基础餐食（1-15级）
  - 各类原型专属餐食（副厨、炊事长、简餐快厨、酿酒大师）
- **背景 fluff 补充**：在主类文件中增加了 `classFluff` 数组，保留了职业的背景描述、角色创建建议及快速建卡指南。

## 3. 翻译与映射修复
- **Source 统一**：确保所有实体的 `source` 字段统一为 `Cook`，`className` 统一为 `厨师`。
- **标签修正**：修复了文档内的嵌套标签，如将 `{@classFeature Cook's Bag...}` 统一映射为本地化的 `{@classFeature 厨包|厨师|Cook|1}`。
- **DC 接口**：为“餐食”设置了正确的 `abilityDc` 接口（基于体质，Con），以适配自动化的 DC 计算。

- `02_子职业/`: 各原型数据文件，包含专属特性正文。
