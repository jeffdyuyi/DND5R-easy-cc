# 圣武士职业数据模块化 - 修改备忘录

## 变更概述
为了对齐双版本（2014/2024）角色构建器的架构要求，本次对 `07_圣武士` 文件夹下的数据进行了深度清理与模块化重构。

## 主要修改
1.  **核心职业特性拆分**：
    *   创建 `01_职业特性/圣武士特性-phb.json`：存放 2014 版圣武士所有的核心职业特性数据。
    *   创建 `01_职业特性/圣武士特性-xphb.json`：存放 2024 版（XPHB）圣武士所有的核心职业特性数据。
2.  **子职业数据模块化**：
    *   将所有独立子职业从 `paladin.json` 的顶层数组中移出，按 `[中文名]-[source].json` 命名规则存放在 `02_子职业/` 文件夹内。
    *   包含：破誓者 (DMG)、王冠之誓 (SCAG)、征服之誓 (XGE)、救赎之誓 (XGE)、守望之誓 (TCE)、以及 2024 版的四种誓言 (XPHB)。
3.  **主文件瘦身 (Registry Index)**：
    *   `paladin.json` 现在仅保留 2014 与 2024 职业的基础定义（生命骰、熟练项、起始装备、法术位表等）。
    *   `classFeatures` 改为引用字符串格式，通过 Hydration 逻辑动态加载详细数据。
    *   `subclasses` 列表仅保留注册信息（Stub），不再嵌套具体特性数据。

## 数据校验与统一
*   **术语对齐**：根据 2024 新规则统一了 `圣武斩 (Paladin's Smite)`、`弃绝众敌 (Abjure Foes)` 等新特性的翻译。
*   **格式规范**：移除了多余的 `subclassFeature` 嵌套，统一使用 `features` 数组结构，确保 UI 渲染一致性。
*   **来源核对**：所有 XPHB 内容已标注 `source: "XPHB"`， classic 内容标注 `source: "PHB"` 或相应扩展源。

## 待办事项 / 注意点
*   2014 版 PHB 的三个子职业（奉献、古贤、复仇）目前在项目源码中为 Stub 状态，依赖底层 Copy 机制。若后续需要本地化完全闭环，需补充其 Full Data 文件。
*   `荣耀之誓 (TCE)` 2014 版在原始数据中被 XPHB 覆盖，已按 Stub 处理，确保不引入冗余错误。
